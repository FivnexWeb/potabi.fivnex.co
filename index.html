About Potabi Systems

What is Potabi?

Potabi is an independently produced operating system. Potabi aims to bring an alternative to the common Linux systems, using the BSD philsophy. Designed to be secure, fast, and simple. Developers and enterprises want less bloated, more secure, and better managed and developed operating systems. Linux operating systems no longer offer this, and it's gotten so bad, it's down to the kernel. The Linux kernel adds two million lines of code per year, or about one Jurassic Park's worth of code every single year. That opens up a lot of potential security holes.

Open Source Software

Potabi is, is is built on, 100% open-source software. This is no different to how Linux functions. This means you can look at all thirteen million lines of code within the potabi-src repository. This is the thirteen million lines of code for the underlying system for all parts of the operating system, including the server, mobile, and desktop editions. Also, yes, you can just run this underlying system, add your own improvements without having to spend hours (maybe even days or weeks) adding the same small improvements for every edition we release.

Multiple System Variants

Unlike many of the BSD branded operating systems - of which we derive from FreeBSD - as well as the majority of Linux distributions, we don't design one operating system variant. We have our server variant, designed and built specifically for server systems. We have our desktop variant, designed and built specifically for the needs of desktop operating system users.

Our goal with providing multiple variants for server and desktop users is to try and actually give users, developers, and enterprises the option to run Potabi as they see fit. Having a singular operating system to manage is so much more simple than having two, three, or maybe thirty five depending on the type of person you are.

Unlike many Linux systems, the server edition we provide is not just our desktop version, with the desktop environment removed, or even just rebranded. We felt like taking the easy way out would be harmful towards the developers we aim to help. We designed server for servers, and desktop for desktops.

13,000,000 vs 19,000,000

Every Linux system runs on the Linux kernel. The Linux kernel has a grand total of nineteen million lines of code, with a Jurassic Park's worth added every year. Yes, Potabi has thirteen million for the underlying system, but unlike Linux, the thirteen million lines of code includes more than the kernel. It also includes many applications, the actual operating system, userland, init systems, everything. In contrast, the Linux kernel's nineteen million lines belong only to the kernel itself, and that's just one of many issues. Also, if you want to check yourself.

Our Mission...

... is to bring an actual alternative operating system that doesn't sacrifice speed, security, or ease of use; an alternative operating system that isn't bloated, elitist, ignorant, or arogant. We don't support the idea of being a gnukkit and to not give users the idea that we are the only hope for people tired of the monopolies and duopolies of the operating system markets. We acknolegde our issues, are honest about our limitations, are fair and kind to those who use operating systems that break our philosphy. We aren't here to hate Linux, the other BSD branded systems, Windows, or macOS. We are here to spark change in the way we see fit. Open-source software exists to be open, be fair, and improve. Having alternatives and competition allows for new ideas to better grow and fester. Even if you are a die-hard Linux/Windows/macOS/other operating system user, please join us.
